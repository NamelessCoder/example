FluidTYPO3: Getting started

Before you continue you should change the backend layout to a single column in full width - that will make it easier to read this help text and others like it. A few ways you can do this:

  • Define mod.SHARED.colPos_list = 0 in pageTSconfig, OR:
  • Create a Backend Layout record with one column and select it in the Appearance tab of this or a parent page, OR:
  • Edit this or a parent page and select FluidTYPO3 Example: Single Column in the Page Layouts tab and select Fluidpages as value of the Backend Layout selector in the Appearance tab of the same page.

    • Continue reading below once you have done this. Note that each of the methods above are mutually exclusive which means that using one means you cannot use the others (due to the way TYPO3 handles backend layouts).

When you've set this up you are ready to view the explanations which are for the most part available as new content types that you insert.

What to do next?

  1. To view examples of page templates and backend layouts integrations you must use the third method from the backend layout selection methods above, then select in Page Layouts the example you want.
  2. To view examples of how custom content elements work simply add a new content element in any page - the available examples are nicely grouped and each have a description. Insert the element to activate the example and study the Plugin options area for how to use that example.
  3. To view examples of how general Flux features work (Flux is the basis of FluidTYPO3 features with the exception of VHS) make instances of content elements from the Flux Examples tab in the new content wizard, like you would do to demonstrate the content element feature.
  4. To view examples of backend module implementations and secondary features you must edit the extension configuration of this extension (key: example) by clicking the little gear icon next to it in the Extension Manager. The extension configuration allows toggling examples which cannot or should not be "always on"; what those features do is described next to each toggle.
