Form field mapping

This element contains an example of mapping Flux form fields to records' fields. When you save this element the value of the field below gets saved into the "subheader" record property (which you should see just above this text).

This happens because the field is named tt_content.subheader in the Flux field definition - the prefix matches the table name and the field matches the column name so internally Flux will copy this value.

This approach only supports simple field value copying (and can also update fields which are not shown as long as the user may change them). For an example about how to manipulate values with more control see the "ContentProvider: Record Updating" example element.
